    文字起こし

    日本語（固定）

    「確定」で Whisper を選んだ場合、停止後に音声全体をこのモデルで文字起こしします。大きいほど高精度で、初回はダウンロードに時間がかかります。WebGPUなら turbo でも高速です。

    マイク

    録音に使うマイクを選びます。選択はこの端末に保存され、次回以降の録音に使われます。

    この画面を開いている間、マイクの入力レベルを表示します。話すとバーが動けば正しく拾えています。緑〜黄が適正、赤が続くときはマイクから離れるか音量を下げてください。

    マイクを利用できません。ブラウザのマイク権限を許可してください（HTTPS または localhost が必要です）。

    AI連携

    これを設定すると、議事録画面の 「AIで議事録を作成（自動）」 が使えます。録音音声をこのキーで直接 Gemini に送り、議事録＋メールを自動生成します。キーはこの端末内（localStorage）にのみ保存され、他人には共有されません。→ 無料のAPIキーを取得（Google AI Studio）

    録音音声と一緒に共有される指示文です。音声から議事録＋メール文面（件名/本文）まで作らせます。このあとに会議名・日付・参加者が自動で付きます。

    この指示文のあとに「会議名・日付・文字起こし」が自動で付きます。用途に合わせて自由に調整できます。

    このアプリについて

    録音・文字起こし・議事録化・出力をすべてブラウザ内で完結するPoCサンプルです。音声は外部へ送信されません。文字起こしは Transformers.js の Whisper を利用しています。

    初回はモデル（数十MB〜数百MB）のダウンロードが発生します。Wi-Fi環境での初回利用を推奨します。

    使い方マニュアル

    NOTELOOP は、会議の録音 → 文字起こし → 議事録づくり → 共有を 1 つの画面でつなぐアプリです。録音ボタンを押すだけで始められます。以下は各機能の使い方です。

    はじめの4ステップ

    1. 1
      録音する

      「録音」画面の中央下にある丸いマイクボタンをタップ。マイクの使用を許可すると録音が始まり、話しながら文字起こしが画面に表示されます。

    2. 2
      停止する

      もう一度ボタンをタップで停止。文字起こしが整い、下部に「録音した音声」の再生・保存が表示されます。誤変換のループは自動で除去されます。

    3. 3
      議事録を作る

      「議事録へ」または録音ボタンから議事録画面へ。「コピーしてClaudeを開く」で会議情報＋文字起こしをコピーし、開いた Claude に貼り付け（⌘/Ctrl+V）→送信すると高精度な議事録ができます。結果は下の欄に貼り付けると、メールや書き出しに使えます。タイトルは内容から自動で付きます。

    4. 4
      出力・共有する

      txt / Word / メールで書き出し、または Claude に渡してさらに高精度な議事録に。一覧に保存で履歴に残せます。

    画面の説明

    • 録音中央のマイクボタンで録音の開始／停止。録音中は経過時間と波形、ライブ文字起こしを表示。「編集」から会議情報（タイトル・日付・参加者）を入力できます。
    • 議事録録音した音声の再生・保存、議事録本文（要点／決定事項／ToDo）、Claude 連携、書き出し、メール作成をまとめた画面です。
    • 履歴保存した議事録をカードで一覧。カードをタップで開き、右下のアイコンで削除・音声保存。最大10件を保存します。
    • 設定文字起こしエンジンの切替、実行バックエンド（WebGPU / WASM）、高精度モデル（tiny〜large-v3-turbo）、録音中の画面常時オン、Claude の指示テンプレートを設定できます。
    • メニュー左上の三本線（PC では常時左側）から各画面へ移動。上部に現在のバージョンを表示します。

    録音中の表示と、確定文字起こしの仕組み

    「録音中のライブ表示」と「停止後の確定」は役割が分かれています（設定で切替）。

    役割使う技術特徴

    記録MediaRecorder音声を m4a で保存。ライブと並行。
    確定・議事録（推奨）音声 → Gemini停止後に「音声をAIに送る」。音声から直接、高精度な議事録＋メールを作成。

    Claude 連携で高精度な議事録に

    1. 1

      議事録画面の下部「コピーしてClaudeを開く」をタップ。会議情報＋文字起こしが自動コピーされ、Claude が開きます。

    2. 2

      開いた Claude の入力欄に 貼り付け（⌘/Ctrl+V）→ 送信 するだけ。

    3. 3

      Claude が要点・決定事項・ToDo を整理した議事録を作成します。

    渡す指示文は 設定 → Claude連携 で編集できます。Claude はチャット上で音声を直接処理できないため、渡すのは文字起こしテキストです（音声は「録音した音声」から別途保存）。

    書き出し・メール

    • ファイル議事録を .txt / .md / Word（.docx） で保存できます。
    • メール議事録から本文を自動作成し、既定メーラー / Gmail / Outlook / .eml / 本文コピー で送信・書き出し。Gmail・Outlook は Web 作成画面が件名・本文入りで開きます。
    • 録音音声音声を保存（m4a 等）と WAVに変換。AI への共有や記録用に使えます。
    • 履歴に保存「一覧に保存」で議事録を履歴に残し、あとから開けます。

    便利な機能

    • 用語の修正・辞書文字起こし上部の「用語修正」で、会社名など誤変換されやすい語を一括置換。辞書に登録すると録音後に自動でチェックします。
    • 会議情報タイトル・日付・参加者（部署＋氏名）を録音の前・最中・後いつでも入力でき、出力にも反映されます。
    • 録音中の通知録音中は通知を表示。画面を消していても通知の「停止」から録音を止められます（対応環境）。
    • 画面の常時オン設定でオンにすると、録音中に画面が消えません（バッテリー消費は増えます）。
    • アプリのインストール（PWA）ブラウザのメニューから「アプリをインストール／ホーム画面に追加」で独立アプリとして起動できます。新しい版は自動で反映されます。

    データとプライバシー

    • 処理場所録音音声・文字起こしはブラウザ内で処理し、アプリ側のサーバへは送信しません。
    • 保存先議事録・設定は端末の localStorage、録音音声は IndexedDB に保存されます（この端末内のみ）。
    • Web Speech 選択時認識のため音声がブラウザ経由でクラウド（Chrome=Google / Edge=Azure）へ送られます。外部送信を避けたい場合は Whisper を選んでください。

    困ったとき（FAQ）

    録音中に文字起こしが表示されない

    マイクの許可を確認してください。Web Speech はインターネット接続が必要です。うまくいかない場合は設定でエンジンを切り替えるか、ページを再読み込みしてください。

    同じ言葉が繰り返される・精度が低い

    マイクに近づき、はっきり話すと改善します。Whisper では高精度モデル（small / large-v3-turbo）に上げてください。反復は自動である程度除去されます。

    画面を消すと録音が止まる

    スマホのブラウザは仕様上バックグラウンドで止まることがあります。設定の「録音中は画面を常時オン」を有効にすると安定します。

    最新版に更新されない

    通常は自動更新されます。反映されない場合はページを再読み込み（2回）するか、ブラウザのサイトデータを消去してください。メニュー上部のバージョン表示で確認できます。

    Word 出力ができない

    Word 出力は外部ライブラリを使うため、オンライン環境で再読み込みしてからお試しください。

    対応環境・注意

    • 推奨ブラウザChrome 系を推奨。マイク利用には HTTPS（または localhost）が必要です。
    • 外部ライブラリWord 出力・Whisper は CDN からライブラリを取得します（初回はオンラインが必要）。
