# NOTELOOP — 議事録自動化Webアプリ（PoCサンプル）

ブラウザだけで完結する議事録自動化アプリです。**録音 → その場で文字起こし → 議事録に整形して編集 → txt / Word / メール下書きで出力** までを、サーバやGPUなしで動かせます。

> これは「実装設計資料（方式A）」を個人が手元で体感するための **PoC（試作）サンプル** です。
> 文字起こしはブラウザ内Whisper、議事録化は簡易ロジック（スタブ）で代替しています。本番の重い部分（サーバ側 faster-whisper / Claude Code ヘッドレス）は後から差し替える前提です。

---

## 目的

添付「実装設計資料」の**方式A**（ブラウザ録音 → 文字起こし → 議事録化 → 出力）を、
サーバ・GPUを立てずに**クライアント完結**で一通り動かせる状態にすること。

## 特長

- 📲 **アプリとしてインストール可能（PWA）**: ホーム画面/デスクトップに追加して、独立したアプリのように起動できます（下記「アプリとしてインストール」参照）。
- 🇯🇵 **日本語専用**: 文字起こし言語は日本語に固定（設定不要）。
- 📱 **メニュー形式のUI**: 左上のハンバーガー（三本線）から**左スライドのドロワーメニュー**で **録音 / 議事録 / 履歴 / 設定** を切り替え。ホームは**中央下部に固定のマイクボタン**と点滅する案内だけのミニマル構成。録音を始めると**上部にウェーブアニメーション**、その下に文字起こしを表示。白ベースの見やすいデザイン。
- 🎙 **録音**: ブラウザの `MediaRecorder` / `getUserMedia`。経過時間・波形・録音の再生。
- 🔀 **文字起こしエンジンを選択可**: **ブラウザ内Whisper**（オフライン・非公開）と **Web Speech API**（無料・リアルタイム高精度 / 要ネット・クラウド送信）を設定で切り替え。
- 🎯 **2パスの高精度文字起こし（Whisper時）**: 録音中は軽量モデル（tiny/base）で**暫定表示**、**録音停止後に音声全体を高精度モデル（base/small/medium）で再処理**して確定版に置き換え。Whisperの30秒コンテキストを活かすため、細切れ処理よりも大幅に精度が向上します（遅い代わりに高精度）。[Transformers.js](https://github.com/xenova/transformers.js) で実行し、**音声は外部へ送信されません**。
- 🤖 **Claude連携（1クリック）**: 会議名・日付・文字起こし・下書きを組み込んだ**プロンプトを自動でクリップボードにコピーし、Claude（アプリ / Web版）を開きます**。あとは貼り付け（⌘/Ctrl+V）→送信するだけで、高精度な議事録が得られます。指示テンプレートは設定で編集可能。
- 📝 **議事録化・編集**: 文字起こしを **要点・見出し / 決定事項 / ToDo** に整形。画面上で直接編集でき、再生成も可能。
- 📤 **出力**: `.txt` / `.md` / `.docx`（Word, [docx](https://github.com/dolanmiu/docx) をブラウザで利用）。
- ✉️ **メール作成**: 議事録（アプリ生成 or Claude生成を貼り付け）からメールを作成し、**既定メーラー（Thunderbird等）/ Gmail / Outlook のWeb作成画面 / .eml** で件名・本文入りで開けます。
- 🔤 **用語の確認・修正**: 会社名など誤変換されやすい語を**一括置換**（辞書に登録すると録音後に自動チェック→ポップアップ）。
- 🔘 **段階変化する録音ボタン**: 録音 → 文字起こし中 → 議事録作成 → メール、と状況に応じてアイコン・動作が変化。
- 🎧 **音声ファイル出力（AI共有向け）**: 録音した音声をダウンロード。**m4a に対応したブラウザでは自動的に .m4a**（AIへの共有に扱いやすい圧縮形式）で保存し、非対応ブラウザでは最適な形式にフォールバック。加えて、どのAI/ツールでも読める **WAV** にも変換できます。
- 💾 **録音後に自動保存**: 録音を止めると、文字起こし・（Whisper時は）音声・会議情報を**自動で履歴に保存**（最大10件、古いものから削除）。
- 👥 **会議情報の入力**: タイトル・日付・**参加者（部署＋氏名）**を録音の前・最中・後いつでも入力可能（部署は一覧から選択、または手入力で追加）。出力・Claude連携・履歴にも反映。
- 🗂 **過去の議事録一覧**: 議事録は localStorage、録音音声は IndexedDB に保存・呼び出し。履歴カードから音声もダウンロード可能。

### 文字起こしエンジンを選べます（設定 → 文字起こしエンジン）

| エンジン | 精度 / 速度 | プライバシー | 備考 |
|---|---|---|---|
| **Web Speech API**（既定） | ○〜◎ **リアルタイム高精度**・待ち時間ほぼ無し | △ 音声が**ブラウザ経由でクラウドへ送信**（Chrome=Google / Edge=Azure）・**要ネット** | モデルDL不要。マイク競合を避けるため**このモードでは録音音声(m4a)は保存されません**（音声も残したい場合は Whisper） |
| **ブラウザ内Whisper** | モデル次第・停止後に高精度化 | ◎ 端末内で完結・**外部送信なし**・オフライン可 | 初回にモデルDL。停止後に音声全体を再処理（進捗・残り時間を表示）。small/medium で高精度 |

> どちらのエンジンでも、録音音声（m4a等）の保存・Claude連携・各種書き出しはそのまま使えます。**外部送信を避けたい実データはブラウザ内Whisper**、**手軽さ・リアルタイム精度を優先する検証は Web Speech API** が向いています。

### 精度を上げたいときは（ブラウザ内Whisper）

**設定 → 高精度モデル** を大きいものに変えてください（遅くなりますが精度が上がります）。

| モデル | サイズ目安 | 精度 / 速度 |
|---|---|---|
| base | 約80MB | 標準 / 速い |
| **small（既定・おすすめ）** | 約250MB | 高精度 / 遅い |
| medium | 約500MB | 最高精度 / かなり遅い（PC推奨） |

> medium は端末メモリを多く使うため、スマホ（特に低価格機）では動かない場合があります。まずは **small** をお試しください。

**精度が出ないとき（「このように…」等の反復が出る場合）:**
- Whisper は無音・小音量で文字列を反復（ハルシネーション）しがちです。本アプリは反復抑制（`no_repeat_ngram_size`）と後処理で自動的に抑えていますが、**マイクに近づける／端末の録音音量を上げる**と大きく改善します。音量が小さいと警告を表示します。
- 静かな環境で、はっきり話すほど精度が上がります。それでも厳しい場合は高精度モデルを **medium（PC推奨）** に上げてください。

## 使い方

1. 下部メニューの **🎙 録音** 画面で **中央の丸いボタン** をタップ → マイクを許可 → 会議を録音。
   - **設定** で高精度モデルやライブ表示を事前に選べます（初回はモデルのダウンロードが走ります）。言語は日本語固定です。
   - ライブ表示がONなら、録音中に暫定テキストが表示されます。
2. もう一度ボタンをタップして **停止** → 音声全体を高精度で再処理し、確定版の文字起こしに置き換わります（「高精度で文字起こし中…」と表示、時間がかかります）。
3. 必要なら **🎧 音声を保存 / WAVに変換** で録音音声もダウンロードできます。
4. **📝 議事録** 画面で **会議名・日付** を入れて **「議事録を生成」** → 要点 / 決定事項 / ToDo に整形（編集可・再生成可）。
5. **出力**: `.txt` / `.md` / `.docx` / **メール下書き** で書き出し、**💾 一覧に保存** で **🗂 履歴** に残せます。

> 動作確認はマイク録音・WASMの相性から **Chrome系ブラウザ** を推奨します。

### 🤖 Claudeで高精度な議事録に（推奨ワークフロー）

アプリ内の簡易ロジックより高品質な議事録が欲しいときは、**議事録画面の「コピーしてClaudeを開く」** を使います。

1. 録音 →（停止後の高精度文字起こし）→ 議事録画面へ。
2. **「コピーしてClaudeを開く」** をタップ → 会議情報＋文字起こし＋下書き＋指示が**自動でコピー**され、**Claude（アプリ / Web版）が開きます**。
3. 開いた Claude の入力欄に **貼り付け（⌘/Ctrl+V）→ 送信** するだけ。
4. Claude が要点・決定事項・ToDo を整理した高精度な議事録を生成します。

- 渡す内容（指示テンプレート）は **設定 → Claude連携** で自由に編集できます（「既定に戻す」も可）。
- ⚠️ Claude はチャット上で**音声（m4a）を直接処理できない**ため、この機能は**文字起こしテキスト**を渡します。音声ファイルは記録用として「録音」画面から別途保存してください。精度は「文字起こしの質 × 指示テンプレート」で決まります。

## アプリとしてインストール（PWA）

公開URL（HTTPS）を開くと、ブラウザから「アプリ」としてインストールできます。インストールすると、ホーム画面/デスクトップのアイコンから、アドレスバーのない独立ウィンドウで起動します。

- **Android（Chrome）**: メニュー（⋮）→「アプリをインストール」または「ホーム画面に追加」。
- **iPhone/iPad（Safari）**: 共有ボタン → 「ホーム画面に追加」。
- **PC（Chrome / Edge）**: アドレスバー右のインストールアイコン（⊕ / モニタ型）→「インストール」。

インストール後もオフラインで**アプリ画面自体は起動**します（初回に取得済みのモデルはブラウザのキャッシュを使います。未取得のモデルのダウンロードにはネットが必要です）。

> 実体は `manifest.webmanifest` と `sw.js`（Service Worker）による PWA です。

## ローカルでの確認

マイク利用にはHTTPS（または `localhost`）が必要です。リポジトリのルートで簡易サーバを起動してください。

```bash
# Python がある場合
python3 -m http.server 8000
# → ブラウザで http://localhost:8000 を開く
```

```bash
# Node.js の場合（任意）
npx serve .
```

`file://` で直接開くと Web Worker / マイクが動かないため、必ず `http://localhost` 経由で開いてください。

## GitHub Pages で公開

1. リポジトリの **Settings → Pages** を開く。
2. **Build and deployment → Source** を **Deploy from a branch** にし、対象ブランチ（例: `main`）／フォルダ `/(root)` を選択して保存。
3. 数十秒後、`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます（自動でHTTPS）。

> 公開URL: _（Pages有効化後にここへ追記してください）_

ビルド不要の静的サイトです。`.nojekyll` を含めており、Jekyll処理を回避します。パスはすべて相対（`./`）指定なので、プロジェクトページ配下でもそのまま動きます。

### 実機での確認手順

- **スマホ（例: AQUOS wish4 / Android Chrome）**: 公開URLを開く → マイク許可 → 録音〜出力まで一通り。
- **PC（例: Windows 11 / Chrome）**: 同じURLを開く → 2カラム表示で文字起こしと議事録を並べて確認。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | 画面（録音 / 議事録 / 履歴のUI） |
| `styles.css` | スタイル（レスポンシブ・ダークモード対応） |
| `app.js` | 録音・タイマー・波形・議事録化・出力・履歴のロジック |
| `worker.js` | Web Worker。Transformers.js で Whisper をブラウザ内実行（モデルをキャッシュし、ライブ用・高精度用を同時保持） |
| `manifest.webmanifest` | PWA マニフェスト（アプリ名・アイコン・表示形式） |
| `sw.js` | Service Worker（アプリシェルをキャッシュし、インストール可能＆起動高速化） |
| `icons/` | アプリアイコン（192 / 512、通常＋maskable） |
| `.nojekyll` | GitHub Pages の Jekyll 処理を無効化 |

### 本番（方式A）への差し替えポイント

議事録化は `app.js` の **`generateMinutes(transcript)`** に集約しています。
本番ではこの関数を **サーバ側 Claude Code（ヘッドレス）** の呼び出しに置き換えるだけで、
入出力の形（`transcript → { summary, decisions, todos }`）を保ったまま差し替えられます。

同様に文字起こしは `worker.js` に分離しているため、将来サーバ側 `faster-whisper` + WebSocket へ移行しやすい構成です。

## 注意・前提

- **精度**: サンプルは小さいモデル（tiny/base）のため精度は本番より低め。用途は「一連の流れの検証」と割り切ってください。
- **初回ダウンロード**: モデルのダウンロード（数十MB〜）が発生します。以降はブラウザにキャッシュされます。
- **CDN依存**: Transformers.js と docx を CDN から読み込みます（オンライン環境が必要）。
- **メール下書き**: `mailto:` は本文長に制限があるため、長い議事録は末尾が省略されます（全文は txt / Word をご利用ください）。
- **公開範囲**: GitHub Pages は公開URLになります。実会議の実データは載せず、ダミー音声で検証してください。

## 技術メモ

- 文字起こし用の音声は `AudioContext`（16kHz mono）で取得し、数秒ごとに Web Worker の Whisper へ渡して逐次表示します。
- 再生用の音声は `MediaRecorder` で別途保持します（フィードバック防止のため、解析用ノードは gain 0 経由で接続）。
- 本資料・実装は 2026年7月時点の一般的理解に基づく検討用です。各ライブラリ・サービスの最新情報と利用規約は実施時にご確認ください。
